Better Together New Features

This is a big one!

Plenty of features were revealed today, both by Tommaso and Jason! Let's start with Jason's feature reveal. If you look at the first two pictures, you will notice stuff. In the first picture, you see snow on the top of the mountains, and grass below. In the second, you see snow and rain along with snow higher in the mountains. This is quite easy to infer: Temperature decreases with height! Now onto Tommaso's features, all version exclusives. This is a bit more confusing for most people, so I'll explain it to the best of my abilities. In the first feature that Tommaso revealed today, he shows the new /tickingarea command: It replaces spawn chunks, and now you can create updateable chunks even when no players are nearby! As long as the chunk limit is not exceeded, feel free to create as many chunks as you desire! The ticking areas work inter-dimensionally, and it enables features such as crop growth, command blocks, redstone, and mob activity even when the player is not around! Keep in mind that this will likely cause lag. There are also variations to the command, which I shall not list here. Next up, we have a tick_world component to entities. It prevents entities from despawning, and it also makes an entity load the world. It also adds an optional despawn distance that is available for entities which should despawn when players are too far away (example used in pic: ender dragon). Just keep in mind that, "[Tommaso] didn't implement this feature at all, [he's] just tweeting about it because the author isn't on twitter :)".

Minecraft Better Together Update revealed!

At the E3 conference earlier today, the Minecraft Better Together update was revealed! It has cross-platform play, and it switches the platforms all over to the Minecraft Bedrock Engine <-- the engine that Pocket Edition uses. This version is Minecraft Pocket Edition's version 1.2. Some pre-confirmed features include parrots, firework stars, firework rockets, music disks, jukeboxes, splitscreen functionality, and coarse dirt. Just a tip so you're not confused like I was, everybody is now using the acronyms BE or BC. BE for Bedrock Engine, or BC for Bedrock Codebase. Use either one!
